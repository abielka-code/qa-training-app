    About / Test Targets (спрощені “вимоги”)

    1. Користувач може “увійти” за email+password. Після входу має відображатися статус входу.
    2. Користувач може додати задачу з непорожнім Title.
    3. Користувач може позначити задачу як виконану (toggle).
    4. Користувач може видалити задачу.
    5. Пошук фільтрує список задач за входженням рядка у Title.
    6. Sync завантажує кілька задач з API та додає їх у список.
    7. Система має показувати коректні повідомлення про успіх/помилку.

    Відомі дефекти (для викладача)

    Деякі дефекти можуть бути увімкнені/вимкнені у config.js. Це потрібно для контрольованого оцінювання.
